Логическое И

Можно группировать операции сравнения с помощью оператора &&, представляющего собой логическое И. В следующем примере, если переменная num больше нуля и одновременно меньше 10, только тогда выведется '+++':

let num = 3;
if (num > 0 && num < 10) {
console.log('+++');
} else {
console.log('---');
}

Задача 1: Проверьте, что переменная num больше нуля и меньше 5.

Задача 2: Проверьте, что переменная num больше или равна 10 и меньше или равна 20.

Задача 3: Проверьте, что переменная num1 равна или меньше 1, а переменная num2 больше или равна 3.

Логическое ИЛИ

Оператор || представляет собой логическое ИЛИ и требует выполнения хотя бы одного условия.

let num1 = 10;
let num2 = -5;

if (num1 > 0 || num2 > 0) {
console.log('+++'); // сработает это
} else {
console.log('---'); }

Приоритет логических операций

Операция && имеет приоритет над ||.

Группировка условий

Хотя операция && и имеет приоритет над ||, часто удобнее использовать группирующие круглые скобки, чтобы явно показать приоритет операций:

let num = 3;

if ( (num > 0 && num < 5) || (num > 10 && num < 20) ) {
console.log('+++');
} else {
console.log('---');
}

Инвертирование логических выражений

Гораздо проще будет использовать оператор !, представляющий собой логическое НЕ. С помощью этого оператора нам достаточно поставить знак ! перед изначальным условием - и оно инвертируется само:

if ( !(num > 0 && num < 5) ) {
console.log('+++');
} else {
console.log('---');
}

Задача 1: Дан следующий код:
if (num1 >= 0 || num2 <= 10) {
console.log('+++');
} else {
console.log('---');
}
Используя оператор ! инвертируйте приведенное условие.

Условия с булевыми значениями

Пусть у нас есть какая-то переменная, которая может принимать значения true или false:

let test = true;

Давайте напишем if, проверяющий нашу переменную на значение true:

let test = true;
if (test === true) {
console.log('+++');
} else {
console.log('---');
}

Задача 2: Проверьте, что переменная test равна true.

Задача 3: Проверьте, что переменная test равна false.

Сравнение значений с булевым типом

В предыдущем примере мы использовали для сравнения оператор ===. В этом случае наша переменная сравнивалась на равенство true как по значению, так и по типу.

Для такого сравнения можно использовать и оператор ==. Если в переменной test всегда будет одно из значений true или false, то ничего и не изменится:

А вот если в переменную test могут попадать любые значения, то все становится намного сложнее.

В таком случае, если в переменной test будет не булево значение, то это значение вначале будет преобразовано к булеву и только затем будет сравниваться.
